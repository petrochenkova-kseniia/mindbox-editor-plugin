# Чеклист качества кейса

Критерии готовности кейса к публикации. Используется на этапе финальной проверки и при анализе пробелов.

---

## Обязательные элементы

Без этих элементов кейс не может быть опубликован.

- [ ] **Цифры до/после** — хотя бы одна ключевая метрика с конкретным значением до и после (не просто «выросло», а «выросло с X до Y»)
- [ ] **Период** — когда начали, когда замерили результат. Ни одна метрика не имеет смысла без временного окна
- [ ] **Что конкретно сделали** — описание механики / инструмента, а не абстрактное «оптимизировали рассылки»
- [ ] **Задача клиента** — зачем это делали, какую проблему решали
- [ ] **Описание компании** — чем занимается, отрасль, масштаб (чтобы читатель мог примерить на себя)

---

## Желательные элементы

Делают кейс сильнее. Если чего-то нет — запросить у клиента.

- [ ] **Контрольная группа** — доказывает инкрементальный эффект. Без нее результат может быть завышен
- [ ] **Скриншоты** — как выглядит письмо, попап, интерфейс программы лояльности, дашборд. Делают кейс нагляднее
- [ ] **Цитата клиента** — живые слова спикера, которые добавляют достоверности
- [ ] **Несколько метрик** — не только главная, но и сопутствующие (например, выручка + OR + отписки)
- [ ] **Описание команды** — кто делал: свои маркетологи, агентство, совместно с Mindbox
- [ ] **Хронология** — таймлайн реализации (что запустили первым, вторым, третьим)
- [ ] **Неудачные попытки** — что пробовали и не сработало. Делает кейс честнее и полезнее

---

## Красные флаги

Сигналы, что с данными что-то не так. Требуют уточнения у клиента.

| Флаг | Что проверить |
|---|---|
| Результат без периода | «Выручка выросла на 30%» — за какой период? YoY, QoQ? |
| Очень высокий результат | ×16 выручки, ROMI 896% — как считали? Есть контрольная группа? |
| Нет связи между действием и результатом | «Запустили рассылки, выручка выросла» — а откуда известно, что выросла из-за рассылок? |
| Цифры не бьются | Доля CRM 30% + доля email 25% + доля пушей 15% = 70% — как так? |
| Сравнение «было/стало» без контрольной группы | Клиенты могли купить и без рассылки |
| Метрики вовлеченности без бизнес-метрик | Высокий OR не равен высокой выручке |
| ROI без учета скидок | Скидки — это расход. Их нужно вычитать |
| Нетипично высокая конверсия | CR 20% из email — точно? Как считали? |
| Выборка < 1000 для AB-теста | Статистически незначимо |
| Нет определения «активного клиента» | «Активных стало больше» — что значит «активный»? |

## Проверка метрик (сверить с метрики-справочник.md)

- [ ] Метрики соответствуют типичным результатам для этой задачи/отрасли
- [ ] Если результат сильно выше типичного — есть объяснение почему
- [ ] Метрики корректно посчитаны (OR от доставленных, а не от отправленных; конверсия с правильной атрибуцией)
- [ ] Указана методология: last click / ассоциированные / инкрементальный подход

---

## Проверка текста

- [ ] Каждый термин объяснен при первом упоминании (если нужно для аудитории)
- [ ] Нет абстрактных формулировок без фактов («значительно выросло», «существенно улучшилось»)
- [ ] Заголовки отражают суть разделов
- [ ] Нет дублирования информации между разделами
- [ ] Логика повествования: от контекста к решению, от решения к результату
- [ ] Текст полезен читателю: можно понять, что сделали и повторить у себя
