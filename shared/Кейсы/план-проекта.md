# План: автоматизация работы над кейсами Mindbox Журнала

## Описание проекта

**Проект:** система автоматизации работы над кейсами Mindbox Журнала в Claude Code.

**Что это:** набор скиллов (команд), файлов знаний и правил, которые вместе образуют пайплайн производства кейса — от первых вводных до готовой статьи.

**Кто пользуется:** авторы Mindbox Журнала. Автор получает от клиента материалы (бриф, презентацию, данные), запускает `/кейс` и проходит все этапы с пошаговым контролем. На каждом этапе Клод делает черновую работу, автор проверяет и направляет.

**Архитектура:**
- **`/кейс`** — скилл-навигатор. Определяет этап, направляет на нужный скилл или выполняет этап сам. Этапы без отдельных скиллов описаны внутри `/кейс`.
- **Скиллы** (команды в `~/.claude/commands/`) — каждый отвечает за один этап. Работают в контексте чата: материалы из предыдущих этапов уже доступны, дублировать не нужно.
- **Файлы знаний** (в папке проекта `Mindbox Журнал/`) — стайлгайд, CRM-знания, каталог кейсов, метрики, типовые структуры, глоссарий, чеклист. Скиллы читают их по мере необходимости.
- **Конституция** (`конституция.md`) — общие правила работы над кейсами: запрет на домыслы, плейсхолдеры вместо выдумки, третье лицо, вопросы спикеру курсивом. `/кейс` читает этот файл первым делом.

**Принципы:**
- Клод не додумывает — если информации нет, честно говорит и формулирует вопрос спикеру
- Каждый скилл делает одну задачу хорошо, а не пытается охватить всё
- Автор контролирует каждый шаг — Клод не перескакивает вперёд без подтверждения
- Вопросы задаются только когда действительно есть неясность, а не для галочки

## Полный пайплайн кейса

```
Вводные (бриф, презентация)
  → 1. Вопросы к интервью
  → 2. Интервью (проводит автор)
  → 3. Расшифровка
  → 4. Карточка кейса
  → 5. Метрики кейса
  → 6. Структура
  → 7. Статья (по разделам)
  → 8. Уточнения → вопросы клиенту
  → 9. Сверка с требованиями заказчика
  → 10. Саморедактура (микроредактор)
  → 11. Финальный чеклист
  → 12. Правки редактора
```

Автор может войти на любом этапе — `/кейс` определит, где он находится, и предложит следующий шаг.
