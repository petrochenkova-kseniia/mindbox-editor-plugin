# Обновление и диагностика плагина mindbox-editor-plugin

Пользователи — редакторы, не разработчики. Все команды выполняешь ты сам. Пользователь не должен ничего вводить в терминал. Говори простым языком, без технических деталей. Если нужно перезапустить Claude Code — скажи об этом понятно.

## Как работает автообновление

Обновление запускается двумя хуками (`hooks/auto-update.sh`, таймаут 15 секунд):

1. **SessionStart** — при каждом запуске новой сессии (без debounce, всегда проверяет)
2. **UserPromptSubmit** — при отправке сообщения пользователем (с debounce: не чаще раза в час)

Скрипт делает следующее:

1. Проверяет debounce (только для prompt_submit): если с последней проверки прошло менее 1 часа — мгновенный выход
2. Записывает timestamp проверки в файл `~/.claude/plugins/.mindbox-last-update-check`
3. Переходит в marketplace-директорию: `~/.claude/plugins/marketplaces/mindbox-editor-plugin`
4. Выполняет `git fetch origin main` — скачивает информацию о новых коммитах
5. Сравнивает локальный SHA с удалённым
6. Если есть обновления — выполняет `git pull --ff-only origin main`
7. Переустанавливает плагин: `claude plugin install` — обновляет кэш
8. Записывает результат в лог: `~/.claude/plugins/logs/mindbox-editor-plugin-update.log`

Весь вывод скрипта подавлен — пользователь ничего не видит.

**Задержка применения:** после обновления кэш обновлён, но Claude Code подхватит новую версию при следующем запуске или после команды `/reload-plugins`. Это нормальное поведение, не ошибка.

## Формат лога

**Путь:** `~/.claude/plugins/logs/mindbox-editor-plugin-update.log`
**Формат строки:** `[YYYY-MM-DD HH:MM:SS] УРОВЕНЬ: сообщение`
**Ротация:** последние 50 строк.

| Уровень | Значение |
|---------|----------|
| `START` | Проверка обновлений начата (указан mode: session_start или prompt_submit) |
| `OK` | Плагин уже актуальный, обновлений нет |
| `UPDATED` | Обновление marketplace успешно применено (указаны старый и новый SHA) |
| `INSTALL` | Начата переустановка плагина |
| `INSTALL_OK` | Плагин переустановлен успешно |
| `INSTALL_ERROR` | Ошибка переустановки (подробности в сообщении) |
| `ERROR` | Произошла ошибка (подробности в сообщении) |
| `SKIP` | Marketplace-директория не найдена, обновление пропущено |

## Порядок действий

Действуй по принципу эскалации: начни с простого, переходи к следующему шагу только если предыдущий не помог.

### Шаг 1. Диагностика

Прочитай лог автообновления:

```
~/.claude/plugins/logs/mindbox-editor-plugin-update.log
```

Объясни пользователю простым языком что произошло:

- `OK: Already up-to-date` — плагин уже актуальный, всё в порядке
- `UPDATED: Updated from ... to ...` — обновление прошло успешно, нужно перезапустить Claude Code
- `ERROR: git fetch failed` — не удалось связаться с сервером (проблема с интернетом или VPN)
- `ERROR: git pull --ff-only failed` — конфликт в файлах, нужно ручное вмешательство (шаг 3)
- `SKIP: Marketplace directory not found` — плагин установлен нестандартно, нужна переустановка (шаг 3)
- Лог-файла нет — автообновление ещё ни разу не запускалось, нужно перезапустить Claude Code

Если проблема понятна и решается перезапуском — скажи об этом и останови. Если нет — переходи к шагу 2.

### Шаг 2. Ручное обновление

Выполни сам:

```bash
cd ~/.claude/plugins/marketplaces/mindbox-editor-plugin && git pull --ff-only origin main
```

Если команда выполнилась успешно — скажи пользователю:
> Обновление загружено. Закройте и снова откройте Claude Code, чтобы изменения вступили в силу.

Если команда завершилась с ошибкой — переходи к шагу 3.

### Шаг 3. Полная переустановка

Если ручное обновление не помогло — переустанови плагин полностью. Выполни команды по очереди:

```bash
claude plugin uninstall mindbox-editor-plugin
```

```bash
claude plugin marketplace remove mindbox-editor-plugin
```

```bash
claude plugin marketplace add petrochenkova-kseniia/mindbox-editor-plugin
```

```bash
claude plugin install mindbox-editor-plugin@mindbox-editor-plugin
```

После переустановки скажи пользователю:
> Плагин переустановлен. Закройте и снова откройте Claude Code. После этого всё должно работать как обычно.

Если и переустановка не помогает — попроси пользователя обратиться к техническому специалисту команды.

## Типичные ошибки и причины

### `git fetch failed (network issue?)`
- Нет интернета или проблема с VPN
- GitHub недоступен (редко)

### `git pull --ff-only failed (not a fast-forward?)`
- В marketplace-директории есть локальные изменения
- Решение: полная переустановка (шаг 3)

### `Marketplace directory not found or not a git repo`
- Marketplace-директория удалена или плагин установлен нестандартно
- Решение: полная переустановка (шаг 3)

### Лог-файл отсутствует
- Автообновление ещё ни разу не запускалось
- Решение: перезапустить Claude Code. Если лог не появился — полная переустановка (шаг 3)
