# Шаблон Pull Request

При создании PR через `gh pr create` использовать следующий формат для body:

## Структура

```
## Что изменено

Кратко описать суть изменений.

## Зачем

Почему нужны эти изменения.

## Тип изменения

- [ ] Новый скилл
- [ ] Обновление существующего скилла
- [ ] Изменение стиля/политики
- [ ] Документация
- [ ] Исправление ошибки
- [ ] Другое (опишите)

## Чеклист

- [ ] SKILL.md содержит корректный frontmatter (name, description, version)
- [ ] Description описывает когда использовать скилл (триггерные фразы)
- [ ] Версия в plugin.json обновлена (если нужно)
- [ ] Изменения не пушатся напрямую в main
```

## Пример команды

```bash
gh pr create --title "feat(skills): добавлен скилл tone-of-voice" --body "## Что изменено

Добавлен новый скилл tone-of-voice с правилами тона коммуникации.

## Зачем

Команде нужны единые правила тона для всех текстов.

## Тип изменения

- [x] Новый скилл

## Чеклист

- [x] SKILL.md содержит корректный frontmatter
- [x] Description описывает когда использовать скилл
- [x] Версия в plugin.json обновлена
- [x] Изменения не пушатся напрямую в main"
```
