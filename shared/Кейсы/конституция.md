# Конституция кейсового пайплайна

Этот файл — общие правила работы над кейсами Mindbox Журнала. Все скиллы кейсового пайплайна читают его.

---

## Пайплайн кейса

```
Вводные (бриф, презентация)
  → 1. Вопросы к интервью
  → 2. Интервью (проводит автор)
  → 3. Расшифровка (/расшифровка)
  → 4. Карточка кейса
  → 5. Метрики кейса
  → 6. Структура (/структура)
  → 7. Статья по разделам (/статья)
  → 8. Уточнения → вопросы клиенту
  → 9. Сверка с требованиями заказчика
  → 10. Саморедактура (/микроредактор)
  → 11. Финальный чеклист
  → 12. Правки редактора (/кейс-ревью)
```

Автор может войти на любом этапе. Оркестратор `/кейс` определяет, где автор находится, и ведёт дальше.

---

## Общие правила

### Запрет на домыслы
Это главное правило. Клод никогда не выдумывает факты, цифры, названия, даты, должности, детали реализации. Если информации нет в материалах:
- Ставить плейсхолдер `[нужно уточнить: что именно]`
- Формулировать вопрос спикеру курсивом: *Спикер, какой был показатель до запуска?*

### Стиль повествования
- Третье лицо: «компания сделала», не «мы сделали»
- Кавычки: «ёлочки» снаружи, „лапки" внутри
- Ё: только при разночтениях («все» vs «всё»)

### Вопросы спикеру
- Выделять курсивом
- Формулировать конкретно — чтобы спикер мог ответить одним сообщением
- Задавать только когда действительно нет информации, а не для галочки

### Иллюстрации
- Предлагать места для скриншотов, схем, графиков
- Формат: `[Иллюстрация: описание, что здесь нужно]`

### Цифры и метрики
- Не менять цифры из материалов
- Если перефразируешь предложение с цифрой — приводить точную цитату из источника в скобках
- Если цифра вызывает сомнение — не исправлять, а помечать вопросом

### Контроль автора
- Не перескакивать вперёд без подтверждения автора
- На каждом этапе останавливаться и ждать проверки
- Предлагать промежуточные проверки для крупных блоков работы

## Карта файлов знаний

Все файлы лежат в `shared/Кейсы/` внутри плагина.

| Файл | Что содержит | Когда использовать |
|---|---|---|
| `style-guide.md` | Редакционные стандарты, стиль, тон | Написание статьи, редактура |
| `crm-знания.md` | Знания о CRM-маркетинге и Mindbox | Все этапы — для понимания предметной области |
| `каталог-кейсов.md` | Список опубликованных кейсов | Вопросы к интервью (поиск аналогов), структура |
| `метрики-справочник.md` | Типичные метрики по задачам | Метрики кейса, вопросы к интервью, уточнения |
| `типовые-структуры.md` | Шаблоны структур по типам кейсов | Структура (как ориентир, не жёсткий шаблон) |
| `глоссарий-mindbox.md` | Термины и их правильное написание | Написание статьи, редактура |
| `чеклист-кейса.md` | Обязательные элементы готового кейса | Уточнения, финальный чеклист |
| `editorial-prompt.md` | 18 редакционных правил (стайлгайд + специфика кейсов) | Редакционная проверка, правки редактора |
